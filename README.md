🩸 Non-Invasive Anemia Detection System

An AI-powered web application that screens for anemia from a photo of a fingernail or eye (conjunctiva) — no blood test required. The system runs five machine-learning/deep-learning models in parallel, combines their outputs, and returns a result along with AI-generated diet and medical guidance.


🎓 Built as a B.Tech final-year project (Information Technology), Sasi Institute of Technology and Engineering, Academic Year 2025–2026.

✨ Features


Non-invasive screening — uses a phone/camera photo instead of a blood draw.
5-model pipeline — XGBoost, SVM, LightGBM, ResNet-50, EfficientNet-B4, and DenseNet-121 (6 models are loaded; the UI highlights 5 at a time depending on version).
Severity estimate — buckets the result into Mild / Moderate / Severe Anemia based on confidence.
Visual confidence chart — auto-generated bar chart comparing every model's confidence score.
AI-generated guidance — Google Gemini produces a structured report: condition overview, precautions, diet recommendations, when to see a doctor, and locally-relevant hospital suggestions for the city the user enters.
Simple web UI — Flask + HTML/CSS, no installation needed for end users; runs from a Google Colab notebook with a public ngrok URL.



🧠 How a prediction is made


Preprocessing — OpenCV detects and crops the region of interest (the nail or eye area) from the uploaded photo, removing background.
Feature extraction — two handcrafted feature sets are built from color (RGB/LAB/HSV), texture (GLCM), and local pattern (LBP) statistics:

58 features → XGBoost
136 features → SVM, LightGBM



Deep learning models — ResNet-50, EfficientNet-B4, and DenseNet-121 take the raw image (resized to 224×224 or 380×380) and predict directly via transfer learning.
Ensemble decision — in the training/evaluation notebooks, the final verdict is a majority vote across the trusted models (ties broken by XGBoost). The project report cites an ensemble accuracy of ~94%, outperforming any single model.
Severity & recommendations — confidence score maps to Mild/Moderate/Severe, and the result + city are sent to Gemini to generate a personalized, structured health report.

⚙️ Setup & Running (Google Colab)

This project is designed to run inside Google Colab (GPU runtime recommended).


Open ANEMIA_FINAL_APRIL.ipynb in Colab.
Mount Google Drive (first cell) and place the 6 trained model files inside:


   /content/drive/MyDrive/Anemia_UI_Final_April/


Run all cells in order. The notebook will:

Install dependencies (flask, torch, torchvision, xgboost, lightgbm, opencv-python, scikit-learn, pyngrok, google-generativeai)
Write app.py, templates/index.html, templates/result.html, and static/style.css to disk
Load all 6 models into memory
Launch the Flask app on port 5000
Open an ngrok tunnel and print a public URL



Open the printed ngrok URL in your browser, upload a fingernail/eye image, enter a city, and click Analyze Now.

⚠️ Important notes before sharing or deploying this publicly


🔑 Hardcoded secrets: the notebook currently has a Gemini API key and an ngrok auth token written directly into the app.py cell. Before pushing this notebook to GitHub or sharing it, remove both and load them from environment variables / Colab secrets instead — anyone with the notebook can currently use your API quota and ngrok account.
Deprecated library: the notebook uses google.generativeai, which Google has end-of-lifed in favor of google.genai. Existing functionality should still run for now, but it's worth migrating.
Not a medical device: this is a screening/awareness tool only and does not replace a clinical blood test (CBC). Results should not be used for diagnosis or treatment decisions.
Dev server: app.run(debug=False) is fine for demos but Flask's built-in server is explicitly not meant for production; use a WSGI server (e.g. Gunicorn) for anything beyond a class demo.



🚧 Limitations (from the project report)


Screening only — cannot replace lab-confirmed diagnosis (CBC).
Nail polish/cosmetics can distort the color features the models rely on.
Accuracy may drop in poor lighting or for very dark skin tones, due to the color-based approach.
Dataset size/diversity is limited, which may affect generalization to new populations.
Binary output only (Anemic / Non-Anemic) — no hemoglobin (g/dL) estimate or numeric severity score in the trained models themselves.
Requires an internet connection (Gemini API + ngrok tunnel).
